Business Process Flows

User Interaction Flow

1. User Authentication
User logs in via OAuth2 (Azure AD / Okta) ‚Üí JWT token issued ‚Üí Session established
2. Tenant Context Resolution
System identifies user's organization ‚Üí Loads tenant-specific configuration ‚Üí Sets data isolation boundaries
3. Intent Detection
User sends message ‚Üí Embedding generated ‚Üí Vector search (FAISS) ‚Üí Azure OpenAI fallback ‚Üí Intent classified
4. Action Routing
Intent mapped to handler ‚Üí Permission check (RBAC) ‚Üí Execute appropriate service (Flow/Connector/Memory)
5. Workflow Execution
Load flow definition ‚Üí Validate connectors ‚Üí Execute steps sequentially/parallel ‚Üí Track progress ‚Üí Handle errors
6. Response Generation
Aggregate results ‚Üí Generate natural language response (LLM) ‚Üí Include context & metadata ‚Üí Return to user
7. Audit & Observability
Log all actions ‚Üí Emit metrics ‚Üí Trace request ‚Üí Update conversation history ‚Üí Store in vector index

Multi-Region Architecture

üåç Active-Active (Global)
  • Multiple regions serve traffic simultaneously
  • GeoDNS routes to nearest region
  • Database replication (async)
  • Global cache synchronization
  • Eventual consistency model
  • Best for: Global user base
  • RPO: Minutes | RTO: Seconds
üîÑ Active-Passive (DR)
  • Primary region handles all traffic
  • Secondary region on standby
  • Continuous backup to secondary
  • Manual or auto-failover
  • Strong consistency
  • Best for: Cost optimization
  • RPO: Hours | RTO: Minutes
